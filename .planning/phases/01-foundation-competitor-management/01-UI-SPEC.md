# Phase 1 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Context note:** No CONTEXT.md exists (discuss-phase skipped). All discretionary decisions below were made by the researcher, documented with rationale, and are consistent with: portfolio-grade live demo, desktop-first, single user/no auth, and an app shell that anticipates the Phase 5 intelligence feed as the eventual money screen.

**Design direction (decision):** Dark-first intelligence dashboard in the Linear/Vercel idiom. Rationale: (1) dark surfaces read as "monitoring/intelligence tooling" and photograph well in a portfolio; (2) the Phase 3/5 diff evidence (green additions / red removals) has maximum legibility on dark neutrals; (3) a single dark theme avoids building and QA-ing a theme toggle in a walking skeleton. Dark-only for v1 — `<html class="dark">`, no toggle.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, badge padding |
| sm | 8px | Compact element spacing, gaps between form fields in a row |
| md | 16px | Default element spacing, card padding, gap between stacked form fields |
| lg | 24px | Section padding, page content padding |
| xl | 32px | Layout gaps, space between page title and content |
| 2xl | 48px | Major section breaks, empty-state vertical padding |
| 3xl | 64px | Page-level spacing (top of empty states on otherwise-blank pages) |

Exceptions:
- **12px** table cell vertical padding (dense data tables; still a multiple of 4)
- **240px** fixed sidebar width; **56px** topbar/page-header height (layout constants, not rhythm tokens)

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Meta | 12px | 400 | 1.5 — timestamps, badge text, helper/error text under inputs, table column headers (uppercase, tracking-wide) |
| Body | 14px | 400 | 1.5 — table cells, form inputs, buttons, nav items, dialog body |
| Heading | 20px | 600 | 1.2 — card titles, dialog titles, section headings |
| Display | 28px | 600 | 1.2 — page titles ("Competitors") |

Rules:
- Exactly these 4 sizes and 2 weights (400, 600). No 500s, no ad-hoc sizes.
- Semibold (600) also permitted at 14px for emphasized inline text (competitor names in table rows) and at 12px for column headers.
- URLs render in **Geist Mono 14px/400**, truncated with ellipsis at container width, full URL on `title` hover.

## Color

Dark-only theme. Values map onto shadcn zinc CSS variables; override in `globals.css`.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #09090B (zinc-950) | App background, main content area |
| Secondary (30%) | #18181B (zinc-900) surfaces with #27272A (zinc-800) 1px borders | Sidebar, cards, table container, dialogs, inputs |
| Accent (10%) | #22D3EE (cyan-400) | See reserved list below |
| Destructive | #EF4444 (red-500) | Delete competitor / remove URL actions only |

Text: primary #FAFAFA (zinc-50), secondary/muted #A1A1AA (zinc-400). Text on accent-filled buttons: #083344 (near-black cyan-950) for contrast.

Accent reserved for:
1. Primary CTA button fill — "Add competitor" page CTA, "Add competitor" add-dialog submit, and "Save changes" edit-dialog submit
2. Active nav item left indicator bar + active nav label
3. Focus rings (keyboard focus-visible)
4. The `changelog`/`pricing` type badges use **muted neutral** styling (zinc-800 bg, zinc-400 text, 12px) — NOT accent. Accent never appears on badges, links, table rows, or icons.

Rationale: cyan is deliberately chosen because Phase 3/5 diff evidence will own green (additions) and red (removals); the accent must never collide with diff semantics. Secondary semantic color: destructive red only.

## Copywriting Contract

Voice: confident, concise, sentence case everywhere (buttons, headings, labels). No exclamation marks. No "please". No generic button labels ("Save", "Cancel", "Submit", "OK") anywhere — every labeled CTA is a specific verb + noun.

| Element | Copy |
|---------|------|
| Primary CTA | "Add competitor" |
| Add-dialog submit | "Add competitor" (matches the page CTA) |
| Edit-dialog submit | "Save changes" |
| Dialog dismissal (add/edit) | No labeled cancel CTA — dismiss via the dialog's X close button and Esc (radix Dialog defaults). X button `aria-label="Close dialog"` |
| Secondary CTA (in add/edit form) | "Add URL" (appends a URL row) |
| Empty state heading | "No competitors yet" |
| Empty state body | "Add a competitor and the changelog or pricing pages you want ProductPulse to watch." + "Add competitor" button |
| Form validation errors (inline, 12px destructive text under field) | Name empty: "Enter a competitor name." / Bad URL: "Enter a full URL starting with http:// or https://." / Blocked host: "This URL points to a private or internal address and can't be monitored." / No URLs: "Add at least one URL to monitor." |
| Save failure (server error) | "Couldn't save competitor. Check your connection and try again." (inline alert above form, not a blank state) |
| Destructive confirmation | Delete competitor: dialog titled "Delete {name}?" body "This removes its monitored URLs and everything ProductPulse has captured for them. This can't be undone." Confirm button (destructive fill): "Delete competitor". Dismiss button (secondary/outline): "Keep competitor". |
| Remove URL (within edit form) | No dialog — inline row removal with icon button (`aria-label="Remove URL"`), reversible until "Save changes" is pressed |
| Success feedback | Toast (sonner): "Competitor added" / "Changes saved" / "Competitor deleted" — noun + past-tense verb, no fluff |

## App Shell & Layout (Phase 1 contract, anticipates Phases 5–6)

Desktop-first; minimum supported viewport 1024px. No mobile work this phase.

**Shell structure:**
- Fixed left sidebar, 240px, zinc-900 bg, zinc-800 right border, full height.
  - Top: wordmark "ProductPulse" (14px semibold) with a small lucide `radar` or `activity` icon at 24px, lg (24px) padding.
  - Nav items (14px, 8px vertical padding, md horizontal): **Feed** (route `/`, placeholder page this phase), **Competitors** (route `/competitors`). Active item: cyan left indicator bar (2px), cyan label, zinc-800 bg pill. Inactive: zinc-400 label, hover zinc-800 bg.
  - Nav order anticipates Phase 5/6 additions (Digest below Competitors later) — leave no visual assumptions that break when items are added.
- Main content area: zinc-950 bg, lg (24px) padding, max content width 1120px, left-aligned (not centered) — dashboard convention.
- Page header inside content: Display-size page title, primary CTA right-aligned on the same row.

**Phase 1 pages:**
1. `/` — Feed placeholder: empty-state layout with heading "Your intelligence feed" and body "Detected competitor changes will appear here once monitoring starts." No CTA to nowhere — link text "Manage competitors →" navigating to `/competitors`. This keeps the demo honest about roadmap without a dead screen.
2. `/competitors` — list + CRUD (contract below).

**Competitor list contract:**
- shadcn Table inside a bordered card. Columns: Competitor (name, 14px semibold), Monitored URLs (stacked rows: type badge + mono URL, one per line), Added (12px muted, relative date), Actions (right-aligned icon buttons: edit `pencil`, delete `trash-2`, ghost variant, destructive hover tint on delete).
- Icon-only action buttons carry aria-labels and shadcn Tooltips: edit → `aria-label="Edit {name}"` (tooltip "Edit"), delete → `aria-label="Delete {name}"` (tooltip "Delete"), remove-URL row button → `aria-label="Remove URL"` (tooltip "Remove URL").
- Add and edit use a shadcn **Dialog** on the list page (per RESEARCH.md Open Question 1 recommendation: less routing surface). Dialog contains: name Input, dynamic URL rows (Select for kind: "Changelog"/"Pricing" + Input for URL + remove-row icon button), "Add URL" ghost button, footer with a single accent submit — "Add competitor" in add mode, "Save changes" in edit mode. Dismissal via the dialog's X close button and Esc; no labeled cancel CTA.
- Delete uses shadcn **AlertDialog**: destructive "Delete competitor" confirm + secondary "Keep competitor" dismiss (copy above).

## Interaction States (contract for executor)

| State | Treatment |
|-------|-----------|
| Form pending | Submit button disabled + lucide `loader-2` spin icon replacing label icon; via `useActionState` pending flag |
| Field invalid | 1px destructive border on input + 12px destructive message below; message from Copywriting Contract |
| Empty list | Centered empty state (2xl vertical padding): 20px heading, 14px muted body, accent CTA |
| Row hover | zinc-900 row bg; action icon buttons visible at all times (no hover-reveal — single-user tool, discoverability wins) |
| Focus-visible | 2px cyan ring, 2px offset, on all interactive elements (shadcn default ring recolored to accent) |
| Loading (RSC navigation) | Skeleton not required this phase; rely on Next.js loading UI only if navigation exceeds ~300ms — optional `loading.tsx` with pulsing zinc-800 blocks |
